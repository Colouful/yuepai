# 约拍用户端产品与 UI 重构审计

## 1. 审计范围

本次审计聚焦 `ruoyi-fastapi-app` 用户端，重点覆盖：

- 首页与发现
- 摄影师、模特、需求、场地与活动
- 发布体系
- 消息与在线沟通
- 订单与履约
- 个人中心与创作者工作台
- 全局视觉体系与移动端适配

## 2. 当前技术栈

- UniApp + Vue 3
- Vite 5
- Pinia / Vuex
- Tailwind CSS + Weapp TailwindCSS
- TypeScript 基础设施已具备，但页面仍以 JavaScript 为主
- 支持微信小程序、H5、App 及其他小程序平台

## 3. 核心判断

当前项目已经具备较完整的路由骨架，但大量页面仍属于静态演示状态，主要表现为：

1. 首页、发现、消息、个人中心等页面大量使用本地静态数组。
2. 作品区域使用纯色渐变块代替真实摄影内容，无法建立摄影平台的视觉可信度。
3. 发布页只有基础输入框，没有表单模型、图片上传、角色需求、时间地点、预算方式和发布校验。
4. 消息列表没有会话跳转、消息分类、未读汇总和业务通知分层。
5. 首页内容层级偏多但缺少主视觉锚点，卡片样式重复，品牌识别度弱。
6. 页面分别使用粉、紫、蓝、绿渐变，缺少统一品牌色和设计令牌。
7. 部分页面存在确定性代码问题，例如个人中心滚动高度引用了不存在的变量。
8. 已有订单、报价、退款、交付、评价等路由，但用户端主链路尚未形成稳定闭环。

## 4. 用户端目标闭环

```text
发现摄影内容/服务
→ 查看创作者主页与作品
→ 私信或发起约拍
→ 提交需求/选择套餐
→ 报价与确认订单
→ 支付与锁定档期
→ 拍摄履约
→ 作品交付与验收
→ 评价、关注与再次预约
```

## 5. 信息架构

底部导航统一为：

1. 首页：推荐、同城、创作者、热门需求、主题内容
2. 发现：作品瀑布流、约拍动态、风格频道
3. 发布：发布约拍、发布作品、发布服务
4. 消息：私信、互动、订单通知、系统通知
5. 我的：资料、订单、收藏、关注、钱包、认证、创作者工具

## 7. 视觉方向

品牌方向：`摄影杂志感 + 年轻创作者社区 + 克制高级感`

### 主色

- 品牌强调色：`#F43F5E`
- 高级黑：`#18181B`
- 暖白背景：`#F7F5F2`
- 内容白：`#FFFFFF`
- 主文本：`#18181B`
- 次文本：`#71717A`
- 边框：`rgba(24, 24, 27, 0.08)`

### 原则

- 图片优先，装饰渐变降级为辅助元素
- 同一页面最多一个主强调色
- 卡片减少阴影，使用边框、留白和层级建立质感
- 统一 16/20/24px 圆角体系
- 统一标题、说明、标签与价格层级
- 统一空状态、加载态、错误态和骨架屏

## 8. 工程改造建议

### 数据层

建议将页面静态数组迁移到：

```text
src/api/yuepai/
src/services/yuepai/
src/store/modules/yuepai/
src/types/yuepai/
```

### 组件层

建议新增：

```text
src/components/yuepai/
├── CreatorCard.vue
├── WorkCard.vue
├── DemandCard.vue
├── ServiceCard.vue
├── SectionHeader.vue
├── FilterBar.vue
├── UploadGrid.vue
├── EmptyState.vue
└── AppSkeleton.vue
```

### 页面状态

所有列表页统一支持：

- 首次加载
- 下拉刷新
- 上拉分页
- 空数据
- 加载失败
- 无网络
- 登录过期
