## 本阶段已完成

### 真实数据与数据库

- 约拍需求表、报名表、报价表、订单表、订单事件表；
- 支付单、支付回调事件、退款单；
- MySQL 初始化脚本；
- Alembic 核心业务迁移和支付迁移；
- 唯一约束、查询索引、乐观锁版本和幂等请求号。

### 需求与报名

- 创建需求草稿；
- 服务端业务校验；
- 提交审核；
- 后台审核权限入口；
- 公开需求分页；
- 数据库关键词、城市、类型、合作方式、角色和排序查询；
- 真实报名人数；
- 重复报名、报名截止、身份和人数上限校验；
- 用户端需求大厅、详情、报名和发布真实接口接入。

### 报价与订单

- 服务端报价创建；
- 报价有效期和双方权限校验；
- 接受报价幂等创建订单；
- 订单号、服务快照、地点快照和联系人快照；
- 订单状态机；
- 买卖双方动作权限；
- 乐观锁防止旧版本覆盖；
- 用户端订单列表和详情真实接口接入。

### 支付与退款

- 微信支付 API v3 请求签名；
- 小程序调起支付参数签名；
- 平台证书验签；
- AES-GCM 回调解密；
- 支付单和事件回调去重；
- 支付金额校验；
- 支付成功后订单事务更新；
- 退款申请、金额和申请期校验；
- 平台审核后发起微信退款；
- 退款回调验签、解密、去重和状态同步；
- 缺少商户配置时明确拒绝，不允许模拟支付成功。

### 生产安全

- API 地址、品牌和协议配置环境变量化；
- 移除默认管理员登录信息；
- 生产启动检查；
- 默认 JWT 密钥、弱数据库密码、模拟支付、本地文件存储、未配置短信和内容审核时阻止生产启动；
- PR 级 Python 编译、Ruff 和参数校验测试。

1. Ruff、单元测试、数据库迁移检查和微信小程序构建全部通过；
2. P0 功能不再使用静态数组或本地存储作为业务事实；
3. 支付和退款在真实微信商户测试环境联调通过；
4. 对象存储、短信、身份核验和内容审核的生产配置齐全；
5. 管理后台能够审核认证、内容、需求、退款和举报；
